# В календарь

Статический генератор файлов `.ics` для добавления событий в Apple Calendar, календарь Android, Google Calendar, Outlook и другие приложения с поддержкой iCalendar.

## Возможности

- заполнение названия, дат, времени, описания, адреса, ссылок, формата, организатора и напоминания;
- предпросмотр события без перезагрузки страницы;
- генерация корректного файла `.ics` прямо в браузере;
- поддержка обычных событий и событий на весь день;
- ссылки поиска адреса в Apple Maps, Google Maps, Яндекс Картах и 2ГИС;
- автоматическое сохранение черновика в текущей сессии браузера;
- русская и английская версия интерфейса;
- светлая, тёмная и системная тема.

## Структура файлов

```text
/
├── index.html
├── styles.css
├── app.js
├── favicon.svg
├── 404.html
└── README.md
```

## Локальный запуск

Откройте файл `index.html` в браузере. Сборка, backend, база данных и внешние API не нужны.

Английская версия доступна через переключатель `RU/EN` или параметр `?lang=en`.

## Хранение данных

Все данные обрабатываются локально в браузере. Черновик формы хранится только в `sessionStorage` текущей вкладки/сессии и очищается при новой сессии. Выбранные язык и тема сохраняются в `localStorage` текущего браузера. Данные не отправляются на сервер.

## Автоматическая публикация через GitHub Pages

В проект добавлен workflow `.github/workflows/deploy.yml`. Он запускается при каждом push в ветку `main` и публикует сайт через GitHub Actions.

1. Создать новый репозиторий на GitHub.
2. Загрузить файлы проекта.
3. Открыть раздел `Settings`.
4. Перейти в раздел `Pages`.
5. В блоке `Build and deployment` выбрать `GitHub Actions`.
6. Сохранить настройки.
7. Сделать push в ветку `main`.
8. Открыть вкладку `Actions` и дождаться завершения workflow `Deploy to GitHub Pages`.
9. Открыть выданную GitHub Pages ссылку.

Workflow публикует только файлы сайта: `index.html`, `styles.css`, `app.js`, `favicon.svg`, `404.html` и `README.md`.

## Ручная публикация через GitHub Pages

1. Создать новый репозиторий на GitHub.
2. Загрузить файлы проекта.
3. Открыть раздел `Settings`.
4. Перейти в раздел `Pages`.
5. В блоке `Build and deployment` выбрать `Deploy from a branch`.
6. Выбрать ветку `main`.
7. Выбрать папку `/root`.
8. Сохранить настройки.
9. Открыть выданную GitHub Pages ссылку.

Сайт использует только относительные пути, поэтому корректно работает по адресу вида `https://username.github.io/repository-name/`.

## Формат ICS

При скачивании создаётся файл MIME-типа `text/calendar;charset=utf-8`. В файл добавляются только заполненные поля события, включая `SUMMARY`, `DESCRIPTION`, `LOCATION`, `URL`, `ORGANIZER` и `VALARM`, если они нужны.
